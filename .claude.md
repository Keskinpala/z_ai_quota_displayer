# Z.ai Monitor - AI Agent Instructions

## 📋 Project Overview

**Z.ai Monitor** is a lightweight Windows desktop overlay widget that displays real-time API quota and token usage for the Z.ai service. Built with Python 3.8+ and Tkinter.

## 🏗️ Architecture

```
z_ai_quota_displayer/
├── app.py          # Main GUI application (Tkinter)
├── config.py       # Configuration management (AppConfig dataclass)
├── zai_client.py   # API client (reusable, standalone)
├── notifier.py     # Custom toast engine (ToastWindow + NotificationEngine)
├── mascot.py       # Zeta animated mascot window
├── test_runner.py  # Local build verification UI
├── requirements.txt # Dependencies (requests, pystray, Pillow)
├── app.spec        # PyInstaller build configuration
├── launch.bat      # Windows background launcher
└── assets/         # Screenshots and resources
```

### 4. notifier.py (Notification Layer)
- **ToastWindow**: Custom bottom-right Tkinter popup; fade in/out, drain-bar, hover-pause, stack-safe
- **NotificationEngine**: threshold checks (hysteresis), periodic interval tick, `update_cfg()` for live reload
- No external dependencies — stdlib only (`tkinter`, `winsound`, `time`, `sys`)
- Severity: info `#4f8ef7`, warn `#f59e0b`, critical `#f4566a`, ok `#22d3a0`
- **Replaces** `win10toast` (fatally broken on Windows 11)

## 🔑 Core Components

### 1. zai_client.py (API Layer)
- **ZaiClient**: HTTP client for Z.ai API
- **LimitInfo**: Quota limit data (TIME_LIMIT, TOKENS_LIMIT)
- **QuotaData**: Complete quota response object
- **Endpoint**: `https://api.z.ai/api/monitor/usage/quota/limit`
- **Auth**: Bearer token in Authorization header

### 2. config.py (Persistence Layer)
- **AppConfig**: Dataclass with all settings
- **Storage**: JSON file at `%APPDATA%/ZaiMonitor/config.json`
- **Fields**: bearer_token, refresh_interval, theme, window_x/y, collapsed, opacity, utc_offset

### 3. app.py (GUI Layer)
- **ZaiMonitorApp**: Main Tkinter application
- **Features**: Draggable window, collapsible UI, dark/light themes
- **Tabs**: Monitor view, Settings panel

## 🔧 Development Guidelines

### Code Style
- Use type hints for all functions
- Follow PEP 8 conventions
- Keep functions focused and small
- Use dataclasses for data structures

### UI Patterns
- All UI redraws happen via `_draw_ui()` method
- Colors are selected dynamically based on quota percentage
- Window position is persisted in config

### Error Handling
- API errors return QuotaData with `error` field set
- Never throw exceptions in API client - return error objects
- Show user-friendly error messages in UI

## 🧪 Testing Changes

1. Run with `python app.py`
2. Test with valid/invalid bearer tokens
3. Test collapse/expand functionality
4. Test theme switching
5. Verify settings are persisted after restart

## ⚠️ Important Notes

- Tkinter is included with Python 3.8+ (no installation needed)
- Only runtime dependency is `requests`
- Config file stores bearer token in plaintext - remind users not to share
- Window uses `-topmost` attribute for always-on-top behavior
- Uses threading for background API calls to prevent UI freeze

### Auto-Refresh Bug (Fixed)
- Root cause: `_manual_refresh()` cancelled pending tick but did NOT reschedule
- Fix: `_manual_refresh()` always calls `_schedule_next()` before returning
- `_auto_tick()` now just fetches + reschedules (no longer delegates to `_manual_refresh`).
